7/** \file
8 * Blob Storage System
9 *
10 * The blob storage system, or the blobstore for short, is a low level
11 * library for placing opaque blobs of data onto a storage device such
12 * that scattered physical blocks on the storage device appear as a
13 * single, contiguous storage region. These blobs are also persistent,
14 * which means they are rediscoverable after reboot or power loss.
15 *
16 * The blobstore is designed to be very high performance, and thus has
17 * a few general rules regarding thread safety to avoid taking locks
18 * in the I/O path. This is primarily done by only allowing most
19 * functions to be called on the metadata thread. The metadata thread is
20 * the thread which called spdk_bs_init() or spdk_bs_load().
21 *
22 * Functions starting with the prefix "spdk_blob_io" are passed a channel
23 * as an argument, and channels may only be used from the thread they were
24 * created on. See \ref spdk_bs_alloc_io_channel. These are the only
25 * functions that may be called from a thread other than the metadata
26 * thread.
27 *
28 * The blobstore returns errors using negated POSIX errno values, either
29 * returned in the callback or as a return value. An errno value of 0 means
30 * success.
31 */
